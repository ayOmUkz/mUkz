# 30-Day Testing Plan

A structured 4-week validation protocol that moves from **paper /
indicator-only** to **live funded execution**. Each week has explicit
KPI gates. **Failing a gate rolls the trader back to the prior week's
protocol for 2 more weeks.**

The plan assumes Phase 1 (manual playbook) and Phase 2 (TV indicator)
are already in place per `automation/automation_roadmap.md`.

---

## Week 1 — Paper Trade, A+ Only

**Objective:** Validate setup detection and discipline on paper.

| Parameter | Value |
|-----------|-------|
| Account type | Paper / Sim |
| Setups allowed | All 5, **A+ grade only** |
| Setups disabled | Any setup with rolling expectancy < 0R (history-based) |
| Size | Mode minimum (1 contract) |
| Max trades / day | 2 |
| Time window | 09:30 – 11:30 ET |
| Indicator | TradingView Phase-2 scanner active |
| Journal | Mandatory per trade + daily |

**KPIs (must hit ≥ 4 of 6 to progress):**

- [ ] ≥ 5 A+ signals taken over the week.
- [ ] Expectancy ≥ +0.5R.
- [ ] 0 off-playbook trades.
- [ ] 100% of trades have completed `setup_review_template.md`.
- [ ] Bias accuracy ≥ 3/5 days.
- [ ] No rule violations.

**Failure protocol:** repeat Week 1 with stricter time window
(09:30 – 10:30 only).

---

## Week 2 — Paper, Add B-Grade at Half Size

**Objective:** Test trade frequency without compromising quality.

| Parameter | Value |
|-----------|-------|
| Account type | Paper / Sim |
| Setups allowed | All 5; A+, A full size; B at half size |
| Size | Mode minimum |
| Max trades / day | 3 |
| Time window | 09:30 – 11:30 ET and 13:30 – 15:00 ET |
| Indicator | TradingView Phase-2 |
| Journal | Mandatory |

**KPIs (must hit ≥ 5 of 7):**

- [ ] ≥ 10 trades total this week.
- [ ] Expectancy ≥ +0.4R.
- [ ] Profit factor ≥ 1.4.
- [ ] Win rate ≥ 40%.
- [ ] B-grade trades' expectancy ≥ 0R (don't let them drag the system).
- [ ] 0 rule violations.
- [ ] 100% of trades have setup reviews.

**Failure protocol:** drop B-grade trades for next 2 weeks; rerun
Week 1 KPIs.

---

## Week 3 — Funded Sim at Mode Sizing

**Objective:** Trade live on the prop firm's sim/eval account with
**full mode sizing**. Real prop-firm rules in force; real DD; real
emotional stakes. Still no live $.

| Parameter | Value |
|-----------|-------|
| Account type | Prop firm **Evaluation** account (or full-rules sim) |
| Setups allowed | All 5; A+, A, B(½) |
| Size | Full Evaluation-mode size (1 contract) |
| Per-trade risk | 0.5% |
| Daily stop | 1.5% (soft), 2.5% (hard) |
| Max trades / day | 3 |
| Time window | 09:30 – 11:30 ET |
| Indicator | Phase-2 scanner |
| Daily protection | All circuit breakers active |

**KPIs (must hit ALL):**

- [ ] No prop firm rule violation.
- [ ] No `LOCKED` state hit.
- [ ] Expectancy ≥ +0.3R.
- [ ] Drawdown buffer remains ≥ 50% all week.
- [ ] Weekly P/L ≥ +0%.
- [ ] 100% of trades journaled.

**Failure protocol:** drop back to Week 2 protocol on the sim
account.

---

## Week 4 — Go / No-Go Evaluation

**Objective:** Decide whether to deploy on a **funded** live account.

This week runs the same protocol as Week 3 BUT with explicit decision
criteria at the end.

| Parameter | Value |
|-----------|-------|
| Account type | Same Evaluation account (continuing from Week 3) |
| Setups allowed | All 5; A+, A, B(½) |
| Size | Evaluation mode |
| Time window | 09:30 – 15:30 ET (test full session) |
| Daily protection | All circuit breakers active |

**Decision criteria for live funded deployment (must hit ALL):**

- [ ] 4-week rolling expectancy ≥ +0.4R.
- [ ] 4-week win rate ≥ 40%.
- [ ] 4-week profit factor ≥ 1.4.
- [ ] 0 rule violations across the 30 days.
- [ ] 0 `LOCKED` sessions in Week 3 + Week 4.
- [ ] 4-week max DD ≤ 3R.
- [ ] Operator can articulate the playbook for any taken trade.
- [ ] Sufficient trade sample: ≥ 30 trades across the 30 days.

**If all criteria pass:** deploy to live funded account at
Evaluation-mode caps for the first 2 weeks; progress to Funded mode
caps only after passing `account_scaling_model.md` § 3 move-up gate.

**If any criterion fails:** stay on sim for another 30 days. Do not
deploy live.

## Continuous Discipline Rules (apply all 4 weeks)

| Rule | Enforcement |
|------|-------------|
| Daily premarket checklist | Mandatory; missing → stand-down day |
| Per-trade `setup_review_template.md` | Mandatory; missing → next-day stand-down |
| Weekly review | Mandatory each Sunday; missing → week's data is "incomplete" and Week 3/4 gates cannot be evaluated |
| Off-playbook trade | One = warning; two in any rolling 4 weeks = restart from Week 1 |
| Manual override of pre-trade gate | Single instance = restart current week from day 1 |

---

## Documentation Output

At the end of the 30 days, produce a single PDF / markdown report
containing:

1. Daily P/L curve.
2. Setup-by-setup expectancy.
3. Rule violations log (target: 0).
4. Bias / regime accuracy.
5. Setups proposed for parameter retuning.
6. Final go / no-go decision with signatures (operator + reviewer).

The report is the artifact that authorizes live funded trading.

---

## What This Plan Does NOT Test

- **Auto-execution.** Phase 4 is not enabled in this plan.
- **Multi-account / multi-strategy.** Single account, single set of
  setups.
- **Off-RTH trading.** ETH/Globex setups are not validated here.

These are tested **after** the 30-day plan passes, in a separate
"Phase 4 Validation" protocol.
